## 关键设计特性

### 1. 频率隔离
- **机械爪**: 200Hz 保证控制精度
- **视觉/触觉**: 30Hz 平衡性能和实时性
- **发布**: 30Hz 匹配最慢的传感器

### 2. 数据解耦
- 使用 ZMQ HWM=1 (高水位标记=1)
- 只保留最新数据,自动丢弃旧数据
- 避免内存积压和延迟累积

### 3. 故障容错
```python
try:
    data = subscriber.subscribeMessage()
except Exception as e:
    # 订阅失败使用上一次的数据或默认值
    # 不会导致进程崩溃
```

### 4. 资源监控
```python
if msg_count % (loop_rate * 2) == 0:
    check_system_resources()  # 检查CPU、内存
    # 每2秒检查一次
```

## 进程生命周期

```
启动阶段:
主进程 ─┬─▶ 进程0 启动 (延迟 0s)
        ├─▶ 进程1 启动 (延迟 1s)
        ├─▶ 进程2 启动 (延迟 2s)
        └─▶ 进程3 启动 (延迟 3s)
             │
             └─▶ 主进程 join(进程3)  # 等待


运行阶段:
所有进程并行运行,通过 ZMQ 通信


终止阶段 (Ctrl+C):
主进程捕获 KeyboardInterrupt
    │
    ├─▶ terminate(进程0) ──▶ 释放CAN资源
    ├─▶ terminate(进程1) ──▶ 释放摄像头0
    ├─▶ terminate(进程2) ──▶ 释放摄像头1
    └─▶ terminate(进程3) ──▶ 关闭ZMQ连接

所有进程优雅退出
```

## 总结

这个多进程架构实现了:
✅ **高频实时控制** (200Hz 电机控制)
✅ **并行传感器处理** (4核同时工作)
✅ **松耦合设计** (ZMQ解耦通信)
✅ **故障隔离** (进程独立,互不影响)
✅ **可扩展性** (易于添加新的传感器或控制器)
